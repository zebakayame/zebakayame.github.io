Ici nous présentons le langage de programmation Java.

Java est un langage de programmation orienté objet qui est créé en 1995 par le designer James Gosling et par l’entreprise Sun Microsystems. Il est constamment mis à jour depuis plus de 27 ans par Oracle et un des langages les plus connus parmi les développeurs. L’un des points positifs de Java aimer par tous est sa portabilité, il est compatible avec presque tous les appareils à l’exception des appareils de chez Apple, sans compter les MacBook qui sont compatibles. De ce fait une application Java requière peu ou quasiment aucune modification pour être importé sur différents appareils qui sont conçus sur différentes architectures ou système d’exploitation. Ceci est grâce à une machine virtuelle Java (JVM) qui peut exécuter tout code Java par les appareils qui le prennent en charge. Ce qui revient donc que si on a envie que des milliers d’application Java soient compatibles avec un appareil spécifique, il suffit donc de développer une seule (le JVM) pour rendre cela possible. Son environnement orienté objet est aussi très facile à manipuler contrairement à certains à ses concurrents tels que le C qui est un langage procédural.

Or il a quelque point faible, contrairement à la plupart des langages de programmation, Java est un langage avec une très lente performance, ceci est dû que la machine n’exécute pas le code directement mais doit en même temps exécuter la machine virtuelle Java qui à son tour doit exécuter le script. Bien que la JVM gère automatiquement la mémoire, ceci peut poser des problèmes si elle n’est quand même pas gérée correctement et peut très vite remplir la mémoire de la RAM ce qui pose problème avec les appareils à performance limitée.

Partons maintenant sur son aspect, et comment il fonctionne voici un bout de code de Java qui affiche sur un terminal le message « Bonjour les gens »

public class application{
    public static void main(String[] args){
        message() ; // appelle de la fonction message()
    }
    public static void message(){ // la fonction message
        System.out.println(“Bonjour les gens”); // code pour imprimer le message
    }
}

Dans ce bout de code, on voit tout d’abord la ligne public class application { Cette classe sert en quelque sorte « stocker » des fonctions à l’intérieur d'elle dont aussi la fonction principale « main » qui peut être aperçue dans la deuxième ligne public static void main(String[] args) { cette fonction sera la première à etre exécuter par le JVM. Tout le code exécuté dépendra principalement de cette classe-ci. Dans la ligne entre les crochets de la classe main, nous voyons la première ligne d’exécution de code, message() ; ce bout de code permet d’appeler la classe nommée message (Attention, à noter que toute ligne de code de Java doit impérativement se terminer avec un : l’exclusion de ceci retournera une erreur au près du JVM) la classe nommée message en question est situé à la ligne 5 du programme. Et elle contient en son interne la ligne de code qui permet d’afficher sur le terminal d’exécution le message Bonjour les gens.
